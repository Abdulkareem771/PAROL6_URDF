# PAROL6 Firmware — Developer Reference

The Teensy 4.1 firmware for PAROL6. Controls 6 stepper-driven joints via closed-loop encoder feedback over USB serial to a ROS 2 hardware interface.

## Architecture

```
src/
├── main.cpp                  ← Entry point: setup(), loop(), ISR
├── hal/
│   ├── LimitSwitch.h         ← Debounced digital input (NPN/PNP/Mechanical)
│   ├── QuadTimerEncoder.h    ← Zero-CPU PWM pulse capture (i.MXRT QuadTimer)
│   ├── SoftwareInterruptEncoder.h  ← Fallback: attachInterrupt() PWM decode
│   ├── FlexPWMGenerator.h    ← Zero-CPU STEP pulse generation (FlexPWM)
│   ├── ToneStepper.h         ← Fallback: software tone() bitbang stepper
│   └── ActuatorModel.h       ← Steps/rad conversion helpers
├── safety/
│   ├── Supervisor.h          ← Safety state machine (NOMINAL/SOFT_ESTOP/FAULT)
│   └── HomingFSM.h           ← SEEKING→BACKOFF→ZEROING→DONE state machine
├── observer/
│   └── AlphaBetaFilter.h     ← Encoder position + velocity estimation
├── control/
│   └── Interpolator.h        ← 1 kHz trajectory interpolation along ROS waypoints
└── transport/
    └── SerialTransport.h     ← USB CDC packet encode/decode
```

## Runtime Flow

```
setup()
  ├── LimitSwitch[6].init()        → pinMode() with correct pull resistor
  ├── HomingSequencer.configure()  → bind velocity + zero-encoder callbacks
  ├── FlexPWMGenerator[6].init()   → start STEP PWM at 0 Hz
  └── QuadTimerEncoder[6].init()   → start PWM capture

loop()  (main thread)
  ├── SerialTransport.process_incoming() → parse <SEQ,pos,vel> commands
  ├── HomingSequencer.tick_1ms()         → drive homing FSM if active
  └── SerialTransport.send_feedback()    → <ACK,seq,pos,vel,lim_state>

run_control_loop_isr()  (1 kHz HW timer)
  ├── QuadTimerEncoder[i].read_angle()   → get motor position
  ├── AlphaBetaFilter[i].update()        → estimate velocity
  ├── LimitSwitch[i].update()            → debounced limit check
  ├── SafetySupervisor.update()          → check velocity limits + watchdog
  └── FlexPWMGenerator[i].set_freq()     → output STEP pulses
```

## Serial Protocol

**Host → Teensy** (sent by `parol6_system.cpp` at `ROS_COMMAND_RATE_HZ`):
```
<SEQ,p1,p2,p3,p4,p5,p6,v1,v2,v3,v4,v5,v6>
```

**Special commands:**
```
<HOME>      → Start homing sequence
<ENABLE>    → Clear SOFT_ESTOP / recover from command timeout
```

**Stale command response** (Teensy → Host, when seq is not newer than last accepted):
```
STALE_CMD
```

**Teensy → Host** (sent at `FEEDBACK_RATE_HZ`):
```
<ACK,seq,p1,p2,p3,p4,p5,p6,v1,v2,v3,v4,v5,v6,lim_state,state_byte>
```
- `p1..p6` — joint positions (rad)
- `v1..v6` — joint velocities (rad/s)
- `lim_state` — bitmask `uint8_t`: bit 0=J1 triggered … bit 5=J6 triggered
- `state_byte` — `uint8_t` enum representing `SafetySupervisor` state: `1`=NOMINAL, `2`=HOMING, `3`=FAULT/ESTOP

**Homing completion strings** (sent to Host after homing sequence):
```
HOMING_DONE
HOMING_FAULT
```

## Configuration

All parameters live in `generated/config.h`, which is **auto-generated by the GUI**. Never edit it manually — use the Firmware Configurator.

Key arrays (`*` = generated but **not yet consumed** at runtime — reserved for future firmware use):

| Array | Purpose |
|-------|---------|
| `STEP_PINS[6]` | Teensy GPIO for stepper STEP signals |
| `DIR_PINS[6]` | Teensy GPIO for stepper DIR signals |
| `ENCODER_PINS[6]` | Teensy GPIO for MT6816 PWM encoder |
| `GEAR_RATIOS[6]` | Mechanical reduction ratios |
| `MICROSTEPS[6]` | Stepper driver microstepping |
| `DIR_INVERT[6]` | Invert STEP direction flag |
| `MAX_VEL_RAD_S[6]` | Per-joint velocity limits used by SafetySupervisor |
| `KP_GAINS[6]` / `KI_GAINS[6]` | PID gains |
| `AB_ALPHA[6]` / `AB_BETA[6]` | AlphaBeta observer gains |
| `LIMIT_ENABLED[6]` | Enable/disable per-joint limit switch |
| `LIMIT_PINS[6]` | GPIO pins for limit switch inputs |
| `LIMIT_PULL[6]` | 0=INPUT, 1=INPUT_PULLUP, 2=INPUT_PULLDOWN |
| `LIMIT_ACTIVE_HIGH[6]` | true=PNP (trigger=HIGH), false=NPN (trigger=LOW) |
| `LIMIT_POLARITY[6]` * | Raw polarity enum (0=FALLING, 1=RISING) |
| `LIMIT_TYPE[6]` * | Switch type enum (0=NONE 1=MECH 2=NPN 3=PNP) |
| `HOMING_ORDER[6]` | Axis order for homing — wrist axes first |
| `HOME_OFFSETS_RAD[6]` | Physical joint zero after homing (rad) |
| `HOMING_SPEED[6]` | Seek speed (steps/s) |
| `HOMED_OFFSET[6]` | Backoff distance after switch trigger (steps) |
| `STANDBY_POS[6]` * | Post-home hold position (steps) |

## Limit Switch Wiring

| Sensor | Pull Setting | Polarity | Notes |
|--------|-------------|---------|-------|
| Inductive NPN (open-collector) | `INPUT_PULLUP` | `FALLING` | Normal=HIGH, triggered=LOW through optocoupler |
| Inductive PNP (sourcing) | `INPUT_PULLDOWN` | `RISING` | Normal=LOW, triggered=HIGH through optocoupler |
| Mechanical NC | `INPUT_PULLUP` | `FALLING` | COM→Pin, NC→GND |

> **Optocoupler is required for inductive sensors** — they run at 12–24 V and would destroy the Teensy 3.3 V GPIO without isolation.

## Build & Flash

Firmware builds inside the `parol6-ultimate` Docker container:

```bash
# Build only
docker run --rm -v $(pwd):/workspace -w /workspace parol6-ultimate:latest pio run

# Build and flash (Teensy must be plugged in with --privileged)
./flash_teensy.sh
```

Or use the GUI's **⚡ Flash** tab which runs PlatformIO automatically.

## Safety State Machine

```
INIT → NOMINAL ←─── feed_watchdog() (only on fresh valid position commands)
         │
         ├── Command timeout > COMMAND_TIMEOUT_MS → SOFT_ESTOP (recoverable)
         ├── joint_velocity > MAX_VEL_RAD_S → FAULT (latched)
         ├── Limit switch triggered (outside homing) → FAULT (latched)
         └── Homing FSM timeout → FAULT (latched)
```

Send `<ENABLE>` to transition back from SOFT_ESTOP → NOMINAL.  
FAULT state requires a full firmware restart (power cycle or reflash).  
During homing, the supervisor watchdog is fed by the HomingSequencer loop — no separate `<ENABLE>` is needed.

## Homing FSM

```
IDLE → begin() → SEEKING (motor moves at -homing_vel)
                     │
                 limit triggered?
                     │
                 BACKOFF (motor moves at +homing_vel for HOMED_OFFSET steps)
                     │
                 ZEROING (zero encoder + observer + interpolator)
                     │
                 DONE → next axis in HOMING_ORDER
```

Travel exceeds `max_travel` without finding the switch → **FAULT**.
